Welcome

This site hosts the custom report and export bundles developed for Sapling Reports, built for the Follett Aspen Student Information System.

Each bundle is a packaged .zip file that can be imported directly into Aspen via Admin → Tools → Exports/Reports → Import Bundle.

Build Your Own Aspen Reports with AI

This open-source repository contains everything you need to generate and modify custom Follett Aspen X2 reports and exports using Claude Code, without writing code by hand.

Describe what you want in plain English. Claude Code reads the included development guides, writes the Java, JRXML, and XML files, packages the bundle, and commits it to your repo. When something breaks on upload, paste the error and it fixes it.

You don't need to read the guides yourself. They exist so the AI produces correct, working code on the first try.

View Repository on GitHub

What's Included

Contents of the Build Your Own repository
Resource Description
Development Guide 9 detailed guide files covering architecture, database schema, Java data sources, JRXML layout, input definitions, best practices, and 30+ documented troubleshooting issues.
Templates Skeleton starter files (Java, JRXML, Input XML, bundle-definition) to copy and rename when beginning a new report.
Working Examples 3 complete report examples at complexity levels 1, 3, and 4 for structural reference.
DEVResources Export A bundled Aspen export (SR-SYS-SCHEMA) that auto-generates your district's database schema, grade term maps, and schedule term data as CSV. Import it into Aspen, run it, and add the output to your repo so Claude Code knows your custom fields.
CLAUDE.md Files Instructions that Claude Code reads automatically each session, providing full context on report architecture, key conventions, and API patterns.

How It Works

Steps to build a report using AI
Step What You Do
1. Clone & setup Clone the repository, install Claude Code (npm install -g @anthropic-ai/claude-code), and run claude in the project directory.
2. Export your schema Import the included DEVResources export into Aspen and run it to generate your district's field definitions. Add the CSVs to BuildResources/.
3. Describe your report Tell Claude Code what data you need, what filters to include, which Aspen view it runs from, and whether it's a PDF report or CSV export.
4. Upload to Aspen Import the generated .zip bundle via Admin → Tools → Exports/Reports → Import Bundle.
5. Iterate If something breaks, paste the exact error from Aspen's job log. Claude Code diagnoses the issue and rebuilds the bundle.
